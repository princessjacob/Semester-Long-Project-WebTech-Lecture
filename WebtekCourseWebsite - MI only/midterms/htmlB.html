Structural Level Tags

It structure the document instead of the content

1. Basic Structural Tags

  1. <body>

    2. contains all the contents of an HTML document, text, hyperlinks, images, tables, lists, etc.

  2. <head>

    3. container for all the head elements (title for the document, scripts, styles, meta information, etc.)

  3. <html>

    4. it tells the browser taht it is an html document

2. Header Tags

  1. <base>

    2. specify a base URI or URL for relative links

  2. <isindex>

    3. obsolete HTML element that puts a text field in a page for querying the document

  3. <link>

    4. used to link to external style sheets

  4. <meta>

    5. used to specify page description, keywords, author of the document, last modified, and other metadata

  5. <script>

    6. used to define a client-side script (JavaScript)

  6. <style>

    7. used to define style information for an HTML document

  7. <title>

    8. defines a title in the browser toolbar

3. Frame Tags

  1. <frame>

    2. used to divide your browser window into multiple sections

  2. <framset>

    3. collections of frame in the browser window

  3. <noframes>

    4. used to link to a non-frameset version of the web site

Block Level Tags

Insert a line break before and after itself

1. Basic Block Level Tags

  1. <address>

    2. defines the contact information for the author/owner of a document or an article

  2. <blockquote>

    3. used for indicating long quotations

  3. <center>

    4. used to center-align text

  4. <div>

    5. used to group block-elements to format them with CSS

  5. <h1>

    6. used to be the title of a post, to emphasize text on the page. It is also the most important heading

  6. <h2>

    7. used to indicate a heading whose level of importance is exceeded only by h1

  7. <h3>

    8. used to indicate a heading whose level of importance is exceeded by h1 and h2

  8. <h4>

    9. used to indicate a heading whose level of importance is exceeded by h1, h2, and h3

  9. <h5>

    10. used to indicate a heading whose level of importance is exceeded by h1, h2, h3 and h4

  10. <h6>

    11. used to define heading levels (Table of Contents). It is also the least important heading

  11. <hr>

    12. represent a horizontal rule

  12. <isindex>

    13. obsolete HTML element that puts a text field in a page for querying the document. It is also both Structural Level and Block Level Tags

  13. <p>

    14. used to define the HTML paragraph element

  14. <pre>

    15. defines preformatted text

2. List Tags

  1. <dd>

    2. used to describe a term/name in a description list

  2. <dir>

    3. used to list directory titles

  3. <dl>

    4. defines a description list

  4. <dt>

    5. defines a term/name in a description list

  5. <li>

    6. used for specifying a list item in ordered, unordered, directory and menu lists

  6. <menu>

    7. defines a list/menu of commands

  7. <ol>

    8. defines an ordered list, it can be numerical or alphabetical

  8. <ul>

    9. defines an unordered list (bulleted)

3. Form Tags

  1. <fieldset>

    2. used for grouping related form elements

  2. <form>

    3. used to create an HTML form for user input

  3. <legend>

    4. used to define a caption for tag

  4. <optgroup>

    5. used to group related options in a drop-down list

  5. <option>

    6. used within a form for defining options in the drop-down list

4. Table Tags

  1. <caption>

    2. defines a table caption

  2. <col>

    3. specifies column properties for each column within a element

  3. <colgroup>

    4. specifies a group of one or more column in a table for formatting

  4. <table>

    5. defines an HTML table

  5. <tbody>

    6. used to group the body content in an HTML table

  6. <td>

    7. (table data/cell), defines a cell in a table

  7. <tfoot>

    8. used to group footer content in an HTML table

  8. <th>

    9. (table header), defines a header cell in a table

  9. <thead>

    10. used to group header content in an HTML table

  10. <tr>

    11. (table row), defines a row in a table

Text Level Tags

Do not insert breaks before and after themselves, but instead can be inserted inline and flow with the text

1. Text Markup Tags

  1. <b>

    2. specifies bold text

  2. <big>

    3. defines bigger text

  3. <i>

    4. displayed in italic

  4. <s>

    5. specifies text that is no longer correct, accurate, relevant

  5. <small>

    6. defines smaller text

  6. <strike>

    7. defines strikethrough text

  7. <tt>

    8. (Teletype Text), produces an inline element displayed in the browser's default monotype font

  8. <u>

    9. renders text with an underline

2. Phrase Markup Tags

  1. <abbr>

    2. defines an abbreviation or an acronym

  2. <acronym>

    3. used only where the phrase comprises initial letters that make a word

  3. <cite>

    4. defines the title of a work

  4. <code>

    5. defines a piece of computer code

  5. <del>

    6. defines text that has been deleted from a document

  6. <dfn>

    7. represents the defining instance of a term in HTML

  7. <em>

    8. it renders as emphasized text

  8. <ins>

    9. defines a text that has been inserted into a document

  9. <kbd>

    10. defines keyboard input

  10. <samp>

    11. defines sample output from a computer program

  11. <strong>

    12. specifies strong text

  12. <var>

    13. specifies a variable

3. Special Markup Tags

  1. <a>

    2. specifies an anchor

  2. <applet>

    3. used for embedding a Java applet within an HTML document

  3. <basefont>

    4. used to specify a base font for the document to use

  4. <bdo>

    5. specifies the direction of text display

  5. <br>

    6. inserts a single line break

  6. <font>

    7. specifies text font, size, and color

  7. <iframe>

    8. specifies an inline sub window

  8. <img>

    9. specifies an image

  9. <noscript>

    10. defines an alternate content for users tah that have disabled scripts in their browser or have a browser that doesn't support script.

  10. <object>

    11. specifies an embedded object

  11. <param>

    12. specifies a parameter for an object

  12. <q>

    13. specifies a short quotation

  13. <script>

    14. used to define a client-side script (JavaScript)

  14. <span>

    15. specifies a section in a document

  15. <sub>

    16. specifies a superscripted text

  16. <sup>

    17. specifies a subscripted text

4. Client-side Image Map Tags

  1. <area>

    2. specifies an area inside an image map

  2. <map>

    3. specifies an image map

5. Form Control Text Tags

  1. <button>

    2. specifies a push button

  2. <input>

    3. specifies an input field

  3. <label>

    4. specifies a label for a form control

  4. <select>

    5. specifies a selectable list

  5. <textarea>

    6. defines a multi-line text input control

HTML element is composed of a start tag and an end tag. Between the start and end tag will be the content. There are different type of HTML elements, the document elements, body elements, style and appearance elements, image elements, list elements, anchor element which has two types; the links and name targets, and other elements.

Document elements:

  1. html

    2. It means that the file is in html format. <html>, this is how the html tag looks like.

  2. head

    3. The <head> tag will contain the title tag.

  3. title

    4. The <title> tag contains the title of the header of the browser.

  4. body

    5. The <body> tag will contain all the body elements which will contain the content of the web page.

Body elements:

  1. h1, h2, h3, h4, h5, h6

    2. These are the heading elements. It will contain the heading/ sub-headings of the web page.

  2. p

    3. The <p> tag contains the paragraph/s in the document.

  3. br

    4. <br> tag means break or line break, when we use
    tag it doesn’t need to have an end tag. It is used for breaking lines into two parts, the other part will be the new line.

  4. hr

    5. <hr> tag means horizontal rule, it creates a horizontal line across the page

  5. blockquote

    6. <blockquote> tag is used for creating a blockquote element, it is use when quoting a content that is from another source.

  6. pre

    7. <pre> tag means preformatted text, it is mostly used for formatting the spacing, indentation, breaks and etc. of a text.

Style and Appearance Elements:

  1. strong

    2. <strong> is used to emphasize the content of a document, it means that it is important and it will bold the content inside the tag.

  2. em

    3. <em> tag means emphasis, it will italicize the content inside the tag to indicate that it is important.

  3. cite

    4. <cite> tag means citation. It will display the content inside the tag as Italic to indicate that it is the title of the cited work.

  4. code

    5. <code> tag means computer code, it will display the content inside the tag in a fixed font.

  5. b

    6. <b> tag will make the content inside it into a bold text

  6. i

    7. <i> tag italicize the content inside it

  7. u

    8. <u> tag underlines the content inside it

  8. tt

    9. <tt> tag means fixed font, it will display the content inside it with a fixed font like “Courier” it is the same as the pre tag and the code tag.

  9. font

    10. <font> tag will turn the content inside it to what the user want the font size, color, and style it will be displayed.

Anchor Element: Links and Named Targets

  1. a href= “URL”

    2. The <a href = “URL”> tag will display the content inside it into a hyperlink text.

  2. a name = “name”

    3. The <a name = “name”> tag sets an anchor/target where the hyperlink can point to.

Image Element:

  1. img

    2. The <img> tag will contain the path of the image the user wants to display and it doesn’t need to have an end tag.

List Elements:

  1. ul

    2. <ul> tag displays the content inside it into an unordered list and will be displayed with a bullet.

  2. ol

    3. <ol> tag displays the content inside it into an ordered list, whether the user wants it to be a numbered list or in a letter list.

  3. li

    4. <li> tag means list item, it will display the content/item inside in a bullet list or numbered list.

  4. dl

    5. <dl> tag means descriptive/ definition list, it will contain the dt and the dd tag.

  5. dt

    6. <dt> tag means the content it the term in the definition list.

  6. dd

    7. <dd> tag means that the content inside it is the definition of the term in definition list.

Other elements:

  1. <!-- -->

    2. this tag is a comment tag.

Global Attributes

  1. accesskey

    2. Key labels representing keyboard shortcuts and associated with elements. This is commonly used to activate or give focus to elements.

  2. class

    3. A name of a classification for belonging to an element.

  3. contenteditable

    4. Usually, just specified as true or false. It specifies whether the content is editable or not.

  4. contextmenu

    5. Elements associated with the value id menu attribute.

  5. dir

    6. Specifies the directionality of an element's text

  6. draggable

    7. Evaluated to true or false and specifies whether an element is draggable or not.

  7. dropzone

    8. Used to specify contents that can be dropped on an element.

  8. hidden

    9. Specifies that an element no longer exists or unimportant.

  9. id

    10. An element's unique identifier

  10. lang

    11. Used to specify a default or commonly used language for all elements

  11. spellcheck

    12. Evaluated as either true or false, this specifies that the elements are subjected for spelling and grammar check.

  12. title

    13. An advisory information with association with an element.

  13. translate

    14. Evaluated as yes or no, this specifies whether all the attributes values and contents of its children to an element should be translated.

  14. style

    15. Apecifies CSS declarations that affects an element's default design.

  15. tabindex

    16. Shows whether a represented element is focusable or not.

DATA-* ATTRIBUTES

Used to keep custom data taken from the user that should be kept in hidden or private to a network or page. This data can be extracted afterwards and is usually used in the page's Javascript to extend user experience.

i.e. <p data-name = " ">

The data-* attributes consists of two parts:

  • an attribute name that should be in lowercase letters and should be at least one character long from data-*
  • an attribute value that can be any string

ROLE, ARIA-* ATTRIBUTES

These attributes provide accessibility to people with disabilities, user agents with WAI-ARIA capabilities, developers, authoring tools and confromance checkers.

i.e.

  • <aria-live = "assertive">
  • <li-role = "menuitem">

i.e. Related to aria-* attributes

WAI-ARIA - WCAG : This is used for contents to conform the page with certain guidelines.

Roles that are included to make it clearly part of WAI-ARIA Role taxonomy.

  1. application

    2. A region declared as a web application.

  2. banner

    3. A region that is mostly site-oriented content.

  3. complementary

    4. Designed to complement the main content similar to the level of its DOM hierarchy

  4. contentinfo

    5. A region that mostly contains the parent document's information.

  5. form

    6. A collection of items and objects that once combined as a whole can be identified as a form

  6. main

    7. The document's main content.

  7. navigation

    8. A collection of navigational links that a user can visit.

  8. search

    9. A collection of items and objects that once combined as a whole can be identified as a search facility.

RULES OF ARIA USE:

  • Don't repurpose an already built-in native HTML element.
  • Don't change native sematics unless there is a need to do so.
  • All interactive ARIA controls should be accessible by the keyboard.
  • Don't use alternatative aria-* attributes on a visible focusable element.
  • All interactive elements should be assigned an accessible name.
